# 信頼ゲート 監査

rigel の主役は **AI（Gemini）の読み取り精度**。**AI 出力を扱うコードは「誤読を断定しない・スキーマで検証する」が
テストで保証されていなければ未完成。** 課金・プライバシーも同様にコードで守る。

対象: $ARGUMENTS

参照: @docs/開発ガイド/04_検証とCIゲート.md（信頼ゲート） / @docs/mahjong-kifu-app-design.md 4章・7章 / @packages/schema/src/index.ts

## チェックリスト（trust-auditor サブエージェントに委譲してよい）

- [ ] **AI 出力は使う前に必ず Zod 検証**（`AiRiverResponseSchema`/`AiHandResponseSchema`、最後に `KifuSchema`）— テストあり
- [ ] **推測で埋めない** — 読めない牌は `tile: null` + `confidence: 0` でスロット保持、枚数・`order` 連番が壊れない — テストあり
- [ ] **confidence を全牌に**持たせ、低い牌が要確認ハイライト（人手修正の起点）になる — テストあり
- [ ] **混在パーツの JSON 抽出**（Code Execution 使用時）が種類でパーツを仕分けて最終JSONを取り出せる — テストあり
- [ ] **課金カウントは成功時のみ +1**、競合で二重加算・取りこぼしが無い — テストあり
- [ ] **撮影画像を永続化しない**（保存は解析後の `Kifu` JSON のみ）
- [ ] `[未確定]` を確定扱いで実装していない（実機検証で確定し、設計ドキュメントを `[決定]` に更新済み）

## 出力

- 各項目を 充足 / 不足 / 該当なし で判定し、不足には具体的な追加テスト案を示す。
- 指標優先順位「**自信満々の誤読を出さない > 正しく読む > 人が直しやすい > 速い・安い**」に照らして、便利機能が信頼ゲートより先行していないか指摘する。
