# 06. 牌のデザイン（UI 表現）

牌（タイル）の見た目をどう表すか。rigel は「写真→牌譜ドラフトを人が確認・修正する」用途なので、
**①速く見分けられる ②confidence の『要確認』を載せやすい ③RN/Web 両対応** を最優先する。

> 関連: [05 APIアーキテクチャ](05_APIアーキテクチャ.md) / 設計ドキュメント 6章（牌は SVG 描画 `[決定]`）

---

## 方式の比較と決定

| 方式 | 牌らしさ | 装飾(要確認/赤) | RN/Web共有 | コスト |
|---|---|---|---|---|
| テキストラベル（"1萬"） | 低 | 枠のみ | 容易 | 最小 |
| Unicode 牌字（🀇…） | 中 | しづらい・赤ドラ不可 | フォント依存 | 最小 |
| SVG 自前・簡易フェイス | 中〜高 | 容易 | 容易 | 小 |
| **OSS 牌画像セット** ← 採用 | **高** | 外枠/帯で付与 | 容易（画像） | 中 |

**[決定] OSS 牌画像セット = [FluffyStuff/riichi-mahjong-tiles](https://github.com/FluffyStuff/riichi-mahjong-tiles)（CC0）** を採用。
本格的な牌の見た目（筒=丸・索=竹・萬=漢数字）を得られ、CC0 で商用配布も自由。`Front` の上にシンボル画像を
重ねて1牌を描く（symbol と `Front` は同じ `viewBox 0 0 300 400`）。
- web: `apps/web/public/tiles/*.svg`（SVG）を `<img>` で重ねる
- mobile: `apps/mobile/assets/tiles/*.png`（PNG書き出し）を `<Image>` で重ねる（RN は PNG をネイティブ表示。静的 require マップ `components/tile-images.ts`）
- 出典/ライセンスは各 `tiles/NOTICE.md`（CC0）。

> **自前・簡易フェイス（`tileFace`）は残す**。`SimpleTile.tsx`（web/mobile）として予備・軽量表示用に保持。
> confidence ハイライト=赤枠、リーチ=赤帯、読めない牌=`?` のオーバーレイは OSS 画像の上に重ねて付与する。

## 面仕様は共有・描画は各プラットフォーム

UI コンポーネントの共有手段は `[未確定]`（Tamagui / RN Web / 自前）。そこで**牌に限っては**、
「何を描くか」の純粋仕様だけを `@rigel/ui` に置き、描画は各アプリが行う。

- 共有（`packages/ui`）: `tileFace(tile)` → `{ kind, rank?, suit?, red, glyph, color }`、
  パレット `SUIT_COLOR` / `RED_TILE_COLOR` / `REVIEW_COLOR`、`needsReview` / `tileLabel`（aria用）。
- 描画（各アプリ）: `apps/web/components/Tile.tsx`（`<svg>`）、`apps/mobile/components/Tile.tsx`（`react-native-svg`）。
  どちらも `tileFace()` の戻り値だけを見て同じ絵を描く。

```
tile(コード)  ──tileFace()──▶  面仕様(kind/rank/suit/red/glyph/color)
                                   │
                  ┌────────────────┴────────────────┐
            web: <svg>                       mobile: react-native-svg
```

## 視覚スペック（v1）

- **数牌**: 上に数字（大）、下にスート記号（萬/筒/索）。字色は萬=赤茶 `#9b1c1c`・筒=青 `#0b5cad`・索=緑 `#1b7a2f`。
- **赤ドラ**(0m/0p/0s): rank=5・赤 `#e60026`。
- **字牌**: 漢字（東南西北白發中）を中央に。
- **読めない牌**(null): `?` をグレーで。**推測で埋めない**方針の可視化。
- **要確認**(`needsReview`=confidence 低 or null): 枠を赤 `#d10f3a`＋薄赤背景。人の修正導線の起点。
- **リーチ宣言牌**: 牌の下に赤帯（横向き回転は将来の polish）。
- サイズ: 30×40（角丸4）。

## 既知の落とし穴（monorepo）

- web=React19 / mobile=React18.3 で `@types/react` が 2 バージョン同居する。`react-native-svg` が
  hoist された React19 型を拾うと「`Svg` cannot be used as a JSX component（refs 不足）」が出る。
  → ルート `package.json` の pnpm `packageExtensions` で `react-native-svg` に `@types/react` を
  **任意の peer** として宣言し、利用側(mobile=18.3)から解決させて回避している。
